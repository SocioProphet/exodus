# Phases

Exodus is a phased system. It does not treat discovery, preservation, processing, and migration governance as one undifferentiated workflow.

## Phase 0 — Scope and controls

Outputs:
- providers, accounts, and domains in scope
- trust boundaries
- intended open targets
- legal / forensic / migration operating mode selection

## Phase 1 — Crawl, topology, index, metadata

Goal: build a byte-light, decision-grade understanding of the estate before expensive ingestion.

Primary outputs:
- provider topology snapshot
- asset census
- dependency graph
- format openness report
- size and temperature profile
- exportability report

## Phase 2 — Ingestion, archival, and processing strategy

Goal: convert the Phase 1 census into an execution plan with explicit budget envelopes and wave ordering.

Primary outputs:
- ingestion plan
- archival plan
- processing plan
- budget proposal
- phase gate recommendation

## Phase 3 — Preservation and intake

Goal: land raw source artifacts, manifests, hashes, and replayable events.

## Phase 4 — Processing and normalization

Goal: extract and materialize useful derived representations over raw evidence.

## Phase 5 — Dashboard governance and recommendation engine

Goal: measure progress, explain residual lock-in, rank next steps, and monitor drift.

## Phase gates

- Gate 1: discovery complete
- Gate 2: budget proposal approved
- Gate 3: preservation fabric working
- Gate 4: processing fabric working
- Gate 5: dashboard governs the exit
